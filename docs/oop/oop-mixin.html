Mixin

Extend provides a nice way of adding new properties from one object to another, but if you don't want existing properties to be replaced, you can use Truck's $.mixin() method. This takes to arguments: the object to mix in, and the object to mix into. If properties on the source object already exist on the target object, they will be ignored. Only new properties will be added to the target.

Let's take a look at the example from $.Extend, but do a mixin instead.

Below is sam object:

var sam = {
  name: 'Sam',
  job: 'developer',
  hobby: 'twiddling thumbs',
  address: '1234 Lane, SomeWhereVille'
}

And this is the object we want to mix into sam:

var newProps = {
  job: 'cook',
  address: '000 DeadEnd, NoPlace',
  age: 1000
};
// Mixin Sam:
$.mixin(newProps, Sam);

// Sam is now:
// (He's still a develoer)
{
  name: 'Sam',
  job: 'developer',
  age: 32
}

As you can see, because we did a mixin, Sam's job did not change, but he now has an age. Like $.extend, you can define a mixin source inline:

// Mixin Sam:
$.mixin({name: 'Sam', job: 'developer', age: 32}, Sam);

Tab Bar

When your app has three to five main screens, a tab bar is a perfect way to structure your app. Having more than five tabs is too cramped for narrower mobile devices, and there are still a lot of those around. Tab bars look very different on Android, iOS and Windows Phone. On iOS the tabs always have icons and the bar is on the bottom, where as on Android and Windows they do not have icons and the bar is on the top. You set up your tab bar and Truck will take care of making it look properly for each platform. If you are targetting iOS, you will need to provide images for the tabs.

Below is how you would set up a tab bar:

// Options for Tab Bar:
//=====================
var opts = {
   icons: ["music", "pictures", "docs", "recipes", "top_rated"],
   labels: ["Music", "Pictures", "Docs", "Recipes", "Favs"],
   selected: 1,
   screens: ['music', 'pictures', 'documents', 'recipes', 'favorites']
};

// Init Tab Bar:
//==============
var myTabbar = $.TabBar(opts);

In the above options, the icons have are rendered on a the tab as a class that you can use to show an icon as a background image of it icon. Each tab has an icon element. On iOS the label will appear under the icon. On Android and Windows the icons won't be visible and only the labels will be. The selected value tells Truck which tab should be selected at load time. Selection is zero-based, so the first tab is 0, the second tab is 1, etc. Screens are the ids of the screens you want the tabs to show. The tabs will be created in the order of the icons, labels and screens, so check your options to make sure you have everything in the right order. If you want your tab bar to exhibit icons for Android or Windows, you can override the default tab bar behavior by providing a option property showIcons: true. Truck will adust the height of the tab bar to accomodate the icons you provided, just as it does for iOS. Labels will be under their icons.

Tab Bar with Navigation List

If you really, really want a tab bar, but more screens than can fit in the tab bar, you could make the last tab a navigation list the directs the user to the additional screens. To set up our tab bar with a navigation list, just provide the last tab with the name "more". Notice how we've set up our tab bar options:

var opts = {
   icons: ["music", "pictures", "docs", "recipes", "more"],
   labels: ["Music", "Pictures", "Docs", "Recipes", "More"],
   selected: 1,
   screens: ['music', 'pictures', 'documents', 'recipes', 'fruits']
};

In this case, the navigation list will be the final tab. It will have an id of "fruits". Tapping a navigation list item will take the user to a detail screen for that fruit. Read Navigation to learn how to set up navigation lists.

You could also use routing with the tab bar's navigation list. That is also covered by the section on Navigation.

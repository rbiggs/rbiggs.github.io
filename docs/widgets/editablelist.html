Editable List

You may have a list that you want your users to be able to modify in some way, such as change the order or deleting some items. You can provide that functionality with Truck's editable list. This allows you to make a list items movable, or deletable, or both. You set this up when you initialize the list.

You can also bind an editable list to a model. That way, when the user is done modifying the list, its changes will to synched to the model. Furthermore, you can set the model to auto-box, thus preserving the user's choice for the next time they load the app. This is useful where you want the user to set up their preferences for when the load the app.

Below is an example of setting up an editable list:

var myEditList = $.EditList({
  deletable: true,
  movable: true,
  model: PeopleModel,
  modelProp: 'guid',
  element: '#editList',
  view: editListView
});

In this example we set the list items to be movable and deletable. We told the list instance what model it should be bound to. We also provided a modelProp value. This is important because it tells Truck what object property to use to identify objects in the model to compare with the changes made by the user. We also provide the list with the view we used to render the list. Truck will update the view's template to contain the elements that the editable list uses. That way if the list gets rerendered after the final changes, it still contains the elements it needs to function as an edtiable list. When you initialize a list to be editable, Truck injects a number of elements into it that were not there when it was first rendered by the view.

Take a look at the files "editable-list.html" and "editable-auto-box.html" to see how these work.

Router

Intro

Truck has a Router. Truck's approach is different from other routing solutions because of its purpose. Truck is for making hybrid apps. Modile apps don't have a concept of urls. You might get a url to download an app, but you're not getting a url for inside your app. Truck's router there does not use a url. It uses a Rest-like interface for routing. And you can get the current and full path of a drill-down navigation path at any time. You could send this to a remote server for a Rest controller to handle.

By not messing with the url, on mobile devices with back buttons, hitting it will immediate return the user as they would expect. Otherwise the back button would slowly navigate the user back through each url modification.

Truck's router allow you to customize navigation lists, slide out menus and tab bars so that you can load content dynamically based on what the user tapped. Please read the section Use.
